# System Shell

Status: Draft contract for Ordo 0.1.0

The 0.1.0 UI uses the Ordo shell pattern:

```text
Primary rail | Second-column evidence index | Main pane
```

## 0.1.0 Navigation

The first product surface is System.

Second-column System menu:

- Brief;
- Health;
- Backup And Restore;
- Schedules;
- Preferences;
- Events.

The default page is the System Brief. Health is supporting evidence, not the
homepage.

## UX Rule

Brief tells the user what matters. The second column shows evidence and tools.
The main pane lets the user inspect or act. Events prove what happened.

## Mobile Rule

Mobile should preserve the same hierarchy as separate states: menu, selector,
and selected detail. It should not squeeze a desktop dashboard into one narrow
screen.
