# Open Source Business Model

Status: public business model direction, not a complete shipped platform

Ordo is AGPL because the user should be able to inspect, modify, host, and
leave with the system that holds their business memory.

The business model is not hostage SaaS. It is a commons-first appliance model
with optional services, support, hosting, premium capabilities, and future
network participation.

## Core Stance

```text
AGPL appliance ownership
+ managed convenience
+ portable backups
+ governed network
+ premium production tools
= more operating leverage without surrendering business memory
```

Managed hosting can be useful. It should be convenience, not captivity.

## What Studio Ordo Can Sell

Studio Ordo can create value without closing the commons:

- managed hosted trials and hosted production appliances;
- setup, migration, recovery, and support services;
- premium production tools such as media rendering, editing, voice, or video
  workflows;
- governed capability packs and domain workflow packs;
- QA, security, review, and rapid response services;
- training, implementation help, and custom operating playbooks;
- future directory, referral, and A2A network services;
- optional hosted Studio Ordo Prime support and premium-job coordination.

The AGPL appliance remains the foundation. The commercial opportunity is making
it easier, safer, and more valuable to run.

## Why Developers Should Care

AI has made it easier for one developer to attempt serious software. It has not
solved distribution, trust, review, support, security, or durable revenue.

Ordo is a bet that independent developers can make real impact through a shared
appliance commons:

- contribute to the core;
- build governed capabilities;
- package domain expertise as reusable work;
- provide QA, support, and implementation services;
- sell premium tools without taking user data hostage;
- benefit from feedback loops across a network of opted-in appliances.

The goal is more independent winners, not another centralized marketplace where
all value routes back to one platform owner.

## Solo-Developed And AI-Assisted

Ordo began as a solo-developed, heavily AI-assisted project. ChatGPT is used for
planning, strategy, writing, image generation, and product exploration. Coding
agents are used for implementation, QA, landing, and research under explicit
GitHub, test, and evidence rules.

At this stage, the project is being built on a roughly $200/month AI tool
budget. That is part of the thesis: disciplined AI-assisted software
manufacturing can let independent developers attempt infrastructure-scale ideas
before a company exists around them.

The point is not that AI writes perfect software. It does not. The point is that
speed only becomes useful when paired with public evidence, QA, review, and a
business model that lets independent people participate.

## What Ordo Will Not Do

Ordo should not win by:

- locking users into a hosted database;
- hiding export paths;
- selling hidden telemetry;
- making cancellation scary;
- pretending future features are already shipped;
- treating community QA as free labor without benefit;
- exposing private appliance data to a network by default;
- making developers depend on one opaque platform relationship.

The commercial strategy should reinforce the product philosophy: ownership,
portability, evidence, governance, and useful work.
